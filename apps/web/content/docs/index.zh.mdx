description: RAGSpine 是一个离线优先的 Python RAG 引擎，具备确定性双通道检索、溯源、工作流脚手架与可选的 HTTP 兼容面。
---

**RAGSpine** 是一个不绑定框架的后端 RAG 引擎，由普通 Python 函数
与类型化 `Protocol` 组装而成。其结构化通道回答证据为事实表的问题；
其叙事 RAG 通道检索带引用的文本。编排器在二者之间路由，并把
缺乏依据的答案替换为明确的拒答。确定性的 `MockProvider` 使基础路径
无需 API 密钥即可使用。

RAGSpine 现在也包含工作流工具链。`ragspine` 命令可以把自然语言
描述转化为安全的 Dify 0.6 工作流，复用相近的内置模板，序列化 JSON 或 YAML，
并输出带版本的图预览。JSON、YAML 与 TOML 工作流文档都可以归一化为
JSON 兼容的映射。这些兼容工具不会把引擎变成 Dify 或 n8n，
运行时也绝不会下载或执行任何上游市场工作流。

## 强制保证什么

- **绝不输出缺乏依据的数值。** 结构化查询失败会变成 `not_found`，无论模型
  如何行文。
- **来源链路。** 事实与叙事片段保留文档与定位符信息。
- **受限内容隔离。** 受限的子 chunk 在提示词组装前被移除；
  父级/窗口扩展无法把它们重新引入。
- **显式执行。** 工作流脚手架、解析、转换、编译与预览都是
  不执行的操作。Dify 与 n8n 的执行路由默认禁用。
- **可插拔的边缘。** provider、embedding、存储、重排器、分块器、队列、连接器及
  其他集成都经由类型化的缝接入。

## 仓库版本与已发布的包

当前文档跟随仓库的 `main` 源码，其包元数据版本为
`0.11.0`。这个数字是源码树版本，并不代表 PyPI 已经提供 0.11.0。
若需要可复现的已发布行为，请用 `ragspine version` 检查你的包管理器
所选中的版本；跟随未发布的 `main` 文档时请使用可编辑的源码安装。
