---
title: はじめに
description: RAGSpine は、決定論的な二重チャネル検索、出典追跡、ワークフローのひな形生成、およびオプションの HTTP 互換サーフェスを備えたオフラインファーストの Python RAG エンジンです。
---

**RAGSpine** は、通常の Python 関数と型付き `Protocol` で組み立てられた、フレームワーク非依存のバックエンド RAG エンジンです。その構造化チャネルは、根拠がファクトテーブルにある質問に回答し、ナラティブチャネルは引用付きのテキストを検索します。オーケストレーターは両チャネル間をルーティングし、裏付けのない回答を明示的な拒否に置き換えます。決定論的な `MockProvider` により、API キーがなくても基本パスを利用できます。

RAGSpine には現在、ワークフローツールも含まれています。`ragspine` コマンドは、自然言語の説明を安全な Dify 0.6 ワークフローに変換し、近い同梱テンプレートを再利用し、JSON または YAML をシリアライズし、バージョン付きのグラフプレビューを出力できます。JSON、YAML、TOML のワークフロードキュメントは、JSON 互換のマッピングに正規化できます。これらの互換ツールがエンジンを Dify や n8n に変えるわけではなく、ランタイムがアップストリームのマーケットプレイスワークフローをダウンロードしたり実行したりすることは決してありません。

## 強制される事項

- **裏付けのない値は返しません。** 構造化ルックアップが失敗した場合、モデルの文章にかかわらず `not_found` になります。
- **出典の系譜。** ファクトとナラティブスニペットは、ドキュメントおよびロケーターの情報を保持します。
- **制限付きコンテンツの隔離。** 制限付きの子チャンクはプロンプト組み立ての前に除去され、親/ウィンドウ展開によって再導入されることはありません。
- **明示的な実行。** ワークフローのひな形生成、パース、変換、コンパイル、プレビューは、実行を伴わない操作です。Dify および n8n の実行ルートはデフォルトで無効になっています。
- **プラガブルなエッジ。** プロバイダー、埋め込み、ストア、リランカー、チャンカー、キュー、コネクターをはじめとする統合は、型付きの縫い目（シーム）を通じて組み込まれます。

## リポジトリのバージョンと公開パッケージ

現在のドキュメントはリポジトリの `main` ソースに従っており、そのパッケージメタデータは
`0.11.0` です。この番号はソースツリー上のバージョンであり、PyPI がすでに 0.11.0 を提供していることを意味するものではありません。再現可能なリリース済みの動作を確認するには、`ragspine version` でパッケージマネージャーが選択したバージョンを確認してください。未リリースの `main` のドキュメントに従う場合は、editable なソースインストールを使用してください。
