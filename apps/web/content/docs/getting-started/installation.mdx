RAGSpine requires **Python 3.10+**. The current release is **0.3.0**. The core imports zero
third-party SDKs and runs fully offline with a deterministic `MockProvider` — you only pull in
heavier dependencies via the optional extras below. Its one non-optional runtime dependency is
the thin shared family core [`corespine`](https://pypi.org/project/corespine/) (`>=0.1.0`),
which owns the cross-package primitives such as the `LLMProvider` seam and `RateLimitedProvider`.

## Install from PyPI

```bash title="install"
pip install rag-spine
# everything (all the optional extras below):
pip install "rag-spine[all]"
# or with uv:
uv add rag-spine
```

<Callout type="info">
The **distribution name is hyphenated** (`rag-spine`), but the **import name is not**:

```python
import ragspine
```

So you `pip install rag-spine` but `from ragspine.agent.agent import answer_question`.

</Callout>

## Optional extras

The base install gives you the offline core. Add extras for the capabilities you need:

| Extra       | Pulls in                                                    | For                                              |
| ----------- | ----------------------------------------------------------- | ------------------------------------------------ |
| `[service]` | fastapi, uvicorn, httpx, rq, redis                          | the HTTP + async-queue layer                     |
| `[pdf]`     | docling                                                     | digital-PDF table extraction                     |
| `[ocr]`     | paddleocr                                                   | scanned-PDF OCR (Linux + NVIDIA GPU)             |
| `[llm]`     | anthropic, openai                                           | the real Anthropic chat provider + OpenAI embeddings backend (lazy-imported) |
| `[embed]`   | sentence-transformers                                       | real embedding models for the vector channel     |
| `[vector]`  | sqlite-vec, pg8000, qdrant-client                           | persistent vector stores (sqlite-vec / pgvector / Qdrant) |
| `[dev]`     | pytest, reportlab, markdown, pdoc, ruff, mypy, build, twine | tests, fixture generation, docs, lint, packaging |
| `[all]`     | `[pdf,ocr,llm,embed,vector,service]`                        | every runtime capability at once (excludes `[dev]`) |

Combine extras with commas:

```bash title="install with extras"
pip install "rag-spine[service,llm]"
```

<Callout type="warn">
  The `[ocr]` extra targets Linux + NVIDIA GPU. `paddlepaddle-gpu` is CUDA-version-specific and is
  **not** declared by the extra — install it separately per PaddlePaddle's official guidance on your
  target box. The real-OCR integration tests are GPU-gated and skipped elsewhere.
</Callout>

## Install from source

Clone the repo and create an isolated environment with [uv](https://docs.astral.sh/uv/):

```bash title="from source"
git clone https://github.com/VoldemortGin/ragspine.git && cd ragspine
uv venv .venv
VIRTUAL_ENV="$(pwd)/.venv" uv pip install -e ".[dev,service]"
```

<Callout type="info">
  The `VIRTUAL_ENV="$(pwd)/.venv"` prefix is **required**: it tells `uv` to install into the `.venv`
  you just created rather than a system Python it might otherwise discover. This is the canonical
  dev setup — editable (`-e`) with the `[dev,service]` extras.
</Callout>

If you have `make`, the same setup is wrapped as targets (run from the repo root):

```bash title="via make"
make venv        # uv venv .venv
make install     # editable install with [dev,service]
make install-all # editable install with [dev,service,llm,embed]
```

## Offline-first by design

The core depends only on the abstraction, never the SDK. Every external dependency — LLM
provider, embedding backend, reranker, OCR engine, retriever, task queue — is a typed
`Protocol`, lazy-imported and injected at the edges. That means:

- A plain `pip install rag-spine` (no extras) is enough to run the structured channel, the
  agent orchestration, and the end-to-end demo **with no API key and no network**.
- The bundled `MockProvider` is deterministic, so offline runs are reproducible.
- You add real backends (Claude, OpenAI, sentence-transformers, PaddleOCR) only when you
  install the matching extra and inject the provider.
