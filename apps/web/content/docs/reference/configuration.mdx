RAGSpine reads its runtime configuration from two places, both verifiable in code:

- **Environment variables** (`RAGSPINE_*`) — assembled into a frozen `ServiceConfig`
  (`src/ragspine/service/config.py`). Shared by the CLI, the FastAPI app, and the worker.
- **The CompanyProfile TOML** — the company identity, entity synonyms, default geography,
  and external/competitor list. The core hardcodes **no** company; it all comes from here.

## Environment variables

Every value below is read in `ServiceConfig.from_env` (`src/ragspine/service/config.py`).
All are optional — the defaults run the offline, deterministic path with no API key.

### Example

```bash
# Structured channel only, offline mock provider
RAGSPINE_DB_PATH=data/fact_metric.db .venv/bin/python scripts/run_server.py --port 8000

# Add the narrative channel and a real provider through an enterprise gateway,
# with an active 90k tokens/minute client-side throttle
RAGSPINE_DB_PATH=data/fact_metric.db \
RAGSPINE_CHUNK_DB_PATH=data/chunks.db \
RAGSPINE_PROVIDER=anthropic \
RAGSPINE_BASE_URL=https://gateway.internal/anthropic \
RAGSPINE_TOKENS_PER_MINUTE=90000 \
.venv/bin/python scripts/run_server.py --port 8000

# Worker for async ingestion (needs Redis)
RAGSPINE_REDIS_URL=redis://localhost:6379/0 .venv/bin/python scripts/run_worker.py
```

### Opt-in retrieval, eval & graph selectors (0.7.0+)

Releases 0.7.0 / 0.8.0 add a family of **opt-in** capability selectors, each read from a
`RAGSPINE_*` environment variable that mirrors a `make_*` factory. All are **default-off** —
unset, the pipeline runs the offline deterministic default (byte-identical), and the LLM- or
vision-backed ones additionally need their extra and a provider. See the
[Retrieval guide](/docs/guides/retrieval) and [Evaluation guide](/docs/guides/evaluation) for
what each one does.

| Variable                  | Default      | Selects                                                                                    |
| ------------------------- | ------------ | ------------------------------------------------------------------------------------------ |
| `RAGSPINE_RERANKER`       | `none`       | local reranker via `make_reranker` — `cross_encoder` / `ce` / `auto`, or `colbert` / `splade` |
| `RAGSPINE_CHUNKER`        | default      | chunking strategy via `make_chunker` — `layout` / `parent_child` / `sentence_window` / `semantic` |
| `RAGSPINE_CONTEXTUAL`     | `none`       | deterministic Contextual-Retrieval index header via `make_index_text_fn`                   |
| `RAGSPINE_POSTPROCESSOR`  | `none`       | post-retrieval chain via `make_postprocessor` — e.g. `mmr,lost_in_middle`                  |
| `RAGSPINE_CORRECTIVE`     | `none`       | CRAG grade→act retriever wrapper via `make_corrective_retriever`                           |
| `RAGSPINE_QUERY_DECOMPOSE`| `none`       | LLM query decomposition via `make_decomposer`                                              |
| `RAGSPINE_QUERY_TRANSFORM`| `none`       | query transform via `make_query_transform` — `hyde` / `rag_fusion` / `step_back`           |
| `RAGSPINE_ADAPTIVE`       | `none`       | Adaptive-RAG complexity routing via `make_adaptive_decomposer`                             |
| `RAGSPINE_RAPTOR*`        | `none`       | RAPTOR multi-granularity tree via `make_raptor_retriever` / `make_raptor_summarizer`       |
| `RAGSPINE_GRAPH_STORE`    | `in_process` | GraphStore backend via `make_graph_store` — `in_process` / `networkx`                      |
| `RAGSPINE_NARRATIVE_GRAPH`| `none`       | opt-in narrative-GraphRAG via `make_narrative_graph`                                        |
| `RAGSPINE_VISUAL_EMBEDDER`| `none`       | ColPali visual-document retrieval via `make_visual_embedder`                               |

<Callout type="info">
  These knobs come from the quality-depth workstreams (W1–W12). Model overrides
  (`RAGSPINE_COLBERT_MODEL`, `RAGSPINE_SPLADE_MODEL`, `RAGSPINE_COLPALI_MODEL`, …) pick a
  specific pinned checkpoint; the full behavior of each layer is documented in the
  [Retrieval guide](/docs/guides/retrieval).
</Callout>

## CompanyProfile

The CompanyProfile is RAGSpine's "no hardcoded company" seam. The core depends on a
`DomainProfile` loaded from TOML — identity, entity synonyms, geography, display labels,
the external/competitor list, and sensitivity rules all live in config, never in code.

<Steps>
<Step>
Copy the template:

```bash
cp config/company.example.toml config/company.toml
```

</Step>
<Step>
Edit it for your deployment, then point at it with `RAGSPINE_COMPANY_CONFIG=config/company.toml`
(or pass the path to `load_company_profile()` directly).
</Step>
</Steps>

The template (`config/company.example.toml`) is a fully fictional placeholder:

```toml title="config/company.toml"
[home]
company_name = "YourCo"
entity_code = "YOURCO_GROUP"

# Entity synonyms: alias (lowercase / abbrev / local language) -> controlled entity_code
[home.synonyms]
"yourco group" = "YOURCO_GROUP"
"yourco" = "YOURCO_GROUP"
"group" = "YOURCO_GROUP"
"yourco north" = "YOURCO_NORTH"
"north" = "YOURCO_NORTH"

# Entity default geography: entity_code -> geography
[home.geography]
YOURCO_GROUP = "GLOBAL"
YOURCO_NORTH = "NORTH"

# Display names: entity_code -> user-visible name (drives clarification / prompt text)
[home.labels]
YOURCO_GROUP = "YourCo Group"
YOURCO_NORTH = "YourCo North"

# External / competitor entities: alias -> display name.
# A hit refuses and proposes re-asking the home-entity equivalent.
[external_entities]
"globex" = "Globex"
"initech" = "Initech"

# Deterministic sensitivity classification for narrative ingestion.
[sensitivity]
default_level = "INTERNAL"
escalate_unknown_to_restricted = false
restricted_filename_patterns = ["board_minutes", "exec_comp", "performance_review"]
restricted_keywords = ["board_minutes", "exec_comp", "绩效"]
```

### What the profile drives

<Cards>
  <Card
    title="Identity"

  ></Card>
  <Card

  <Card

  ></Card>
  <Card
    title="Sensitivity"
    description="Deterministic RESTRICTED classification rules for narrative ingestion — restricted content is filtered at two retrieval exits before reaching a prompt."
  ></Card>
</Cards>

<Callout type="warn">
  No company name is hardcoded anywhere in the engine. Home identity, entities, the tool schema, and
  refusal text all derive from `load_company_profile()` — see
  `src/ragspine/common/company_profile.py`. Never backfill a real company into code.
</Callout>
