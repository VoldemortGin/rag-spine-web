## Contesto

Dify ГЁ una popolare piattaforma low-code di orchestrazione di applicazioni LLM; i suoi workflow si esportano come
`.yml` (DSL). Molti utenti hanno un workflow / advanced-chat funzionante costruito in Dify ma vogliono
**uscire dal lock-in della piattaforma**: un pezzo di Python puro indipendente dai framework, leggibile, versionabile
ed eseguibile offline. L'identitГ  di RAGSpine ГЁ esattamente il "backend indipendente dai
framework" ([ADR 0009](/it/docs/decisions/adr-0009): nessun lock-in di framework + dipendenze
esclusivamente con licenze permissive), quindi **compilare** un workflow Dify in Python puro imperativo вЂ” distribuendo
**suggerimenti statici di ottimizzazione** lungo il percorso вЂ” si sposa naturalmente con la direzione del prodotto.

Vincoli rigidi: la famiglia ГЁ completamente sincrona; il core resta a zero SDK; le chiamate LLM passano
attraverso la cucitura `corespine.LLMProvider.chat`; il `MockProvider` offline ГЁ il default;
il gate delle sole licenze permissive regge; `mypy --strict`; pydantic solo al confine.

## Decisione

**`.yml в†’ parse в†’ IR в†’ codegen + optimize` вЂ” tre stadi disaccoppiati da un'unica IR de-Dify-izzata.**

- **parse** вЂ” `yaml.safe_load` (PyYAML, MIT, supera il gate delle licenze) legge il documento in
  un dict, validato da modelli di confine pydantic v2 (`extra='allow'` tollera i campi sconosciuti,
  quindi un nuovo campo Dify non rompe mai il parser). pydantic compare **solo in questo stadio**.
  PyYAML ГЁ importato in modo lazy dietro un nuovo extra opzionale `[dify]`; il core `dependencies` resta intatto.
- **IR** вЂ” `DifyDoc в†’ WorkflowIR`. I nodi si normalizzano in sottoclassi `IRNode` di dataclass frozen;
  i riferimenti alle variabili (`value_selector` / `{{#nodeId.field#}}`) si normalizzano in
  `VarRef` / `Literal` / `TemplateValue`; gli archi portano `source_handle`; l'ordinamento topologico di Kahn
  produce `topo_order` e `parallel_layers`; un ciclo solleva `CyclicGraph`. Pura stdlib вЂ”
  zero pydantic, e zero concetti Dify trapelano sotto questo strato.
- **codegen** вЂ” `WorkflowIR в†’ GeneratedCode`. La topologia si appiattisce in uno script imperativo
  `def run_workflow(inputs, *, provider=None) -> dict`; i nodi LLM diventano
  `provider.chat(messages)`; gli strati paralleli usano `concurrent.futures.ThreadPoolExecutor`
  (**non viene generato alcun async** вЂ” la famiglia ГЁ completamente sincrona). Pura stdlib.
- **optimize** вЂ” `WorkflowIR в†’ list[Suggestion]`: **otto regole puramente statiche** (zero chiamate
  API, non tocca mai l'ambiente live; i tetti d'ambiente sono iniettabili) вЂ”
  PARALLEL_001/002, BOTTLE_001/002, CACHE_001, RESOURCE_001/002, LLM_001.

Approda in `src/ragspine/dify/{parse,ir,codegen,optimize}/ + api.py + errors.py`. La facade:
`compile_dify_yaml(source, *, target='ragspine', provider_expr='MockProvider()',
emit_trace=False, analyze=True) -> CompileResult(code, suggestions, ir)`;
`analyze(source, *, env=None) -> list[Suggestion]`; a basso livello
`parse_dify_yaml / lower_to_ir / generate_code`. CLI: `ragspine dify compile <path>`.

## I sei default dell'MVP (В§7 вЂ” questioni aperte, default risolti)

Per far avanzare l'MVP, sei questioni aperte hanno adottato **default ragionevoli** (rivedibili in review):

1. **Runtime di destinazione** вЂ” di default uno script imperativo ragspine puro; un parametro `target=` ГЁ
   riservato come cucitura, con un target di orchestrazione spineagent lasciato per P7.
2. **Ambito dell'ottimizzazione** вЂ” regole puramente statiche (zero API). Profiling dinamico / stima reale dei
   token rimandati.
3. **Nodi `code`** вЂ” **resi inline** come funzioni locali generate, con un commento che segnala
   l'"assunzione di fiducia sulla sorgente" (un nodo code di Dify esegue giГ  codice utente arbitrario; il
   compilatore mantiene lo stesso confine di fiducia e non lo mette in sandbox).
4. **Nodi non supportati** (http-request / tool / knowledge-retrieval / parameter-extractor /
   plugin) вЂ” generano funzioni hook con `raise NotImplementedError` + una docstring dettagliata
   (producono uno **scheletro eseguibile**, mai il fallimento dell'intera compilazione); registrati esplicitamente in
   `GeneratedCode.warnings`.
5. **ModalitГ  dell'app** вЂ” prima `workflow` + `advanced-chat` (con nodi `answer`).
   `conversation_variables` / memoria / stato di conversazione multi-turno lasciati per P7.
6. **Dipendenza YAML** вЂ” un nuovo extra opzionale `[dify]` (PyYAML, `safe_load`); il core
   `dependencies` resta intatto.

## Alternative considerate (respinte)

- **`.yml в†’ code` diretto (senza IR).** Respinta вЂ” i tre stadi sarebbero fortemente accoppiati,
  aggiungere un backend di destinazione (spineagent) o fare analisi statica sarebbe difficile; la IR ГЁ il
  fulcro condiviso per l'ottimizzatore e per i molteplici target.
- **Un interprete a runtime (eseguire il `.yml` al volo).** Respinta вЂ” significherebbe costruire
  un altro mini runtime Dify, vanificando l'intero scopo di compilare in Python puro di qualitГ 
  libreria e abbandonare la piattaforma, e contraddicendo la posizione indipendente dai framework dell'ADR 0009.
- **Generare codice async.** Respinta вЂ” la famiglia ГЁ completamente sincrona;
  `ThreadPoolExecutor` copre il parallelismo ed ГЁ piГ№ leggibile ed eseguibile offline.
- **Mettere PyYAML nel core `dependencies`.** Respinta вЂ” solo lo stadio di parse ne ha bisogno;
  l'import lazy + l'extra `[dify]` mantengono vero il principio "`import ragspine` non trascina alcuna dipendenza pesante".

## Conseguenze

- Un nuovo extra `[dify]` (PyYAML); il core `dependencies` e il core offline restano invariati.
- I nodi non supportati producono uno **scheletro eseguibile + warning** вЂ” gli utenti ottengono uno scaffolding che
  possono completare direttamente, non una compilazione fallita.
- Lo strato IR riserva punti di estensione per target futuri (orchestrazione spineagent, stima reale dei
  token, ottimizzazione dinamica).
- Il record era inizialmente **proposed**: i sei default attendevano la review; ogni modifica segue
  il principio "sostituire, non modificare" (un nuovo record o un cambio di stato).
