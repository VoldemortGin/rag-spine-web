一份**架构决策记录（ADR）**记录一项重大决策：迫使做出该决策的
背景、决策本身、被否决的备选方案，以及随之而来的
后果。RAGSpine 把它的 ADR 作为不可变的历史记录保存。

## ADR 在这里的运作方式

- **一个文件一项决策。** 每份记录是一个单独、自包含的选择
  （仓库中的 `docs/adr/NNNN-kebab-title.md`）。
- **只追加且不可变。** ADR 一旦接受就绝不编辑。要
  推翻一项决策，需要新增一份_新的_ ADR 来替代旧的——历史
  保持完整、可审计。
- **只替代，不编辑。** 这就是为什么一份记录可以描述较早的路径（比如
  `src/` 布局之前的文件位置）而不算"错"：它记录的是
  当时的决策原貌。
- **豁免漂移追踪。** 由于它们是历史记录，ADR 不携带 `covers`
  frontmatter，也不被 `scripts/check_doc_drift.py` 检查。

<Callout title="贯穿主线">
  这些决策反复出现的原则是**把保证与灵活性分开**：
  不变量（防编造机制、溯源、来源隔离、依赖许可证、安全
  门控）是硬性的、确定的、不可插拔的，而其余一切（领域、语言、LLM、
  embedding、重排器、解析策略）是软性的——由配置或 `Protocol` 驱动。
</Callout>

## 记录列表

[ADR 0002](/zh/docs/decisions/adr-0002) 是产品方向的伞形记录；它
索引了在同一场设计访谈中敲定的八份逐项 ADR（0003–0010）。

<Callout type="warn" title="ADR 编号是仓库本地的">
  网站 ADR 0012–0015 保留了较早的家族/网站发布序列。当前的
  RAGSpine 引擎仓库也有自己的 ADR 文件，部分编号相同；例如，
  网站的 ADR 0012 记录的是 corespine LLM 缝，而引擎的
  `docs/adr/0012-onboarding-complexity-budget.md` 是另一项决策。不要仅凭
  编号推断二者相同。下面新增的网站 ADR 0016–0018 是引擎中相同编号
  记录的英文转录。
</Callout>
